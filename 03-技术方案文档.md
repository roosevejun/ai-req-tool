### 2. API与错误码规范
1. 统一响应体
   - 成功：`{ "code": 0, "message": "...", "data": ..., "traceId": "..." }`
   - 失败：`code` 非0，`message` 为可读错误。

2. 错误码建议（示例，需按项目落表/文档）
   - `10001` 参数校验失败
   - `10002` 未认证
   - `10003` 未授权
   - `20001` 生成失败（LLM输出不合规/校验失败）
   - `30001` 业务冲突（如重复幂等键）

### 3. 文档生成实现细则（Agent流程落地）
1. 阶段化生成
   - 阶段A：澄清问题生成（仅输出可落地问题）
   - 阶段B：功能清单生成
   - 阶段C：接口说明生成
   - 阶段D：数据表设计生成
   - 阶段E：验收标准生成
   - 阶段F：设计/技术汇总与一致性检查

2. 结构化草稿（JSON）+ 校验
   - 每个阶段先产出 JSON 草稿
   - 后端使用 JSON Schema/规则引擎校验
   - 校验失败：回到“需要补充信息”或触发重试

3. 文档渲染
   - 将模型输出片段按固定模板拼装为 Markdown
   - 表格字段必须严格匹配约定表头（便于验收与机器校验）

### 4. 数据落库与审计
1. 生成任务表：保存任务状态、输入摘要、每阶段输出摘要、最终输出引用
2. 版本与回溯：保存模板版本、生成参数（如开关项）、输出哈希（可选）
3. 审计字段：`createdAt/createdBy/updatedAt/updatedBy`、`traceId`

### 5. 可观测性与质量控制
- 日志规范：每次任务阶段输出结构化日志（traceId/jobId）
- 指标：耗时、成功率、校验失败类型分类
- 兜底策略：LLM超时/格式错误 -> 返回可读错误 + 必要的澄清问题

- 输入内容敏感词过滤/合规策略（如有）
- 访问控制：项目维度的RBAC（按需）
