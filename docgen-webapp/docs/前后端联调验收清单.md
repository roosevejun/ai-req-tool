# 前后端联调验收清单（项目管理 + 需求管理 + 版本页）

本文档用于你在公司内做联调和验收时快速打勾确认。

## 1. 基础环境

1. 后端服务可访问：`/api/docgen/health` 返回 `UP`
2. 前端可正常打开，接口代理正确
3. PostgreSQL 表已初始化：
- `sys_*`
- `pm_*`
- `rm_*`

## 2. 登录与权限

1. 登录成功返回 token
2. 未登录访问受保护接口返回 `401`
3. 无权限访问返回 `403`
4. Swagger 中：
- 登录接口为公开
- 非公开接口带 Bearer 锁标识
- 接口可见 `x-permissions`

## 3. 系统管理（用户/角色/权限）

1. 用户列表可查询
2. 创建用户成功
3. 更新用户信息成功
4. 重置密码成功
5. 角色列表、创建、更新成功
6. 权限列表、创建、更新成功
7. 用户绑定角色成功
8. 角色绑定权限成功

## 4. 项目管理

1. 项目列表接口返回正常
2. 创建项目成功（`PROJECT:CREATE`）
3. 更新项目成功（`PROJECT:EDIT`）
4. 项目详情可查询

## 5. 需求管理

1. 项目下需求列表可查询
2. 创建需求成功（`REQUIREMENT:CREATE`）
3. 更新需求成功（`REQUIREMENT:EDIT`）
4. 需求详情可查询

## 6. 需求版本页

1. 版本列表可查询（`REQUIREMENT:VERSION_VIEW`）
2. 版本详情可查询
3. 当前版本标识正确（前端高亮）
4. 指定版本 Markdown 下载成功

## 7. AI 需求工作台

1. 创建会话成功（`REQUIREMENT:AI_CHAT`）
2. 会话状态查询成功
3. 对话补充成功，未确认项逐步收敛
4. 生成 PRD 成功（`REQUIREMENT:GENERATE_PRD`）
5. 导出当前会话 PRD 成功

## 8. 关键异常用例

1. 无效 token：返回 `401`
2. 禁用用户：返回 `401`
3. 权限不足：返回 `403`
4. 不存在项目/需求：返回业务错误且可读
5. LLM 不可用：返回可读错误信息（非空白错误）

## 9. 回归建议

1. 后端执行：`mvn test`
2. 前端执行：`npm run build`
3. 数据库执行回滚脚本后可重新初始化：
- `rollback-project-requirement.sql`
- `init-system.sql`
- `init-project-requirement.sql`
