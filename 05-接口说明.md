## 接口说明（通用模板）

**项目名称**：{{项目名称}}  
**版本**：{{版本号}}  
**日期**：{{日期}}  

> 说明：本文件作为后端实现契约来源。所有接口都需输出统一响应体：  
> `{ code, message, data, traceId }`

### 1. 接口约定
1. 认证方式：`JWT`（或按项目选择）
2. 列表接口分页：`pageNo`（默认1）、`pageSize`（默认20）
3. 错误状态码约定：
   - `400`：参数校验失败
   - `401`：未认证
   - `403`：未授权
   - `404`：资源不存在
   - `409`：幂等/业务冲突
   - `422`：生成输入不完整/需澄清
   - `500`：内部错误

### 2. 接口清单
| 接口ID | URL | Method | 认证方式 | 请求参数（Query/Path/Body） | 响应数据（data） | 可能错误码 |
|---|---|---|---|---|---|---|
| TEMPLATE_LIST | `/api/templates` | GET | JWT（可选） | Query：`projectType`(可选) | `templates[]` | 401/500 |
| TEMPLATE_UPDATE_STATUS | `/api/templates/{templateId}/status` | PUT | JWT（必需） | Path：`templateId`；Body：`enabled:boolean` | `success:true` | 400/401/403/500 |
| DOCGEN_CREATE_JOB | `/api/docgen/jobs` | POST | JWT | Body：`projectId, templateVersionId, businessDescription, options?`，可选：`idempotencyKey` | `jobId, status` | 400/401/403/422/409/500 |
| DOCGEN_LIST_JOBS | `/api/docgen/jobs` | GET | JWT | Query：`pageNo,pageSize,status?`，可选：`projectId` | `jobs[] + total` | 400/401/403/500 |
| DOCGEN_GET_JOB | `/api/docgen/jobs/{jobId}` | GET | JWT | Path：`jobId` | `jobDetail`（含阶段状态/结果引用） | 401/403/404/500 |
| DOCGEN_GET_CLAIRFY | `/api/docgen/jobs/{jobId}/clarify` | GET | JWT | Path：`jobId` | `clarifyQuestions[]` | 401/403/404/422/500 |
| DOCGEN_SUBMIT_CLARIFY | `/api/docgen/jobs/{jobId}/clarify` | POST | JWT | Path：`jobId`；Body：`answers{...}` | `status` | 400/401/403/422/500 |
| DOCGEN_GET_DOCUMENT | `/api/docgen/jobs/{jobId}/documents/{docType}` | GET | JWT | Path：`jobId, docType`（docType如：PRD/DESIGN/TECH/FUNC/API/DB/ACCEPT） | `markdown` 或 `documentMeta` | 401/403/404/500 |
| DOCGEN_EXPORT | `/api/docgen/jobs/{jobId}/export` | POST | JWT | Body：`format`（md/doc/pdf）、可选：`docTypes[]` | `downloadUrl` | 401/403/400/404/500 |

### 3. 统一返回结构（示例）
- 成功：
```json
{ "code": 0, "message": "ok", "data": { ... }, "traceId": "..." }
```
- 生成输入不完整（需要澄清）：
```json
{ "code": 20001, "message": "需要澄清信息", "data": { "clarifyQuestions": [...] }, "traceId": "..." }
```
