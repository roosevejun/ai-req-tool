## 数据表设计（通用模板）

**项目名称**：{{项目名称}}  
**版本**：{{版本号}}  
**日期**：{{日期}}  

> 说明：以下表结构为“文档生成 Agent”通用骨架。你可按实际需要增减字段/表。

### 1. 表：`project`
1. 业务含义：项目主数据（用于隔离不同业务域/租户）。
2. 字段建议：
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGINT | PK | 项目ID |
| name | VARCHAR(200) | NOT NULL | 项目名称 |
| type | VARCHAR(50) | NULL | 项目类型（可选） |
| createdAt | DATETIME(3) | NOT NULL | 创建时间 |
| createdBy | BIGINT | NOT NULL | 创建人 |
| updatedAt | DATETIME(3) | NOT NULL | 更新时间 |
| updatedBy | BIGINT | NOT NULL | 更新人 |

3. 索引：
- `idx_project_type`(type)

### 2. 表：`template`
1. 业务含义：文档模板及其版本管理的基础表。
2. 字段建议：
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGINT | PK | 模板ID |
| projectType | VARCHAR(50) | NULL | 适用范围（可选） |
| name | VARCHAR(200) | NOT NULL | 模板名称 |
| description | TEXT | NULL | 模板说明 |
| enabled | TINYINT | NOT NULL | 是否启用 |
| createdAt/updatedAt/... | - | - | 审计字段 |

3. 索引：
- `idx_template_projectType_enabled`(projectType, enabled)

### 3. 表：`template_version`
1. 业务含义：模板版本（用于回溯与稳定生成）。
2. 字段建议：
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGINT | PK | 版本ID |
| templateId | BIGINT | FK | 对应template.id |
| versionNo | VARCHAR(50) | NOT NULL | 版本号（如 v1.0） |
| systemPrompt | MEDIUMTEXT | NOT NULL | Agent系统提示词 |
| userPrompt | MEDIUMTEXT | NOT NULL | 用户提示词模板 |
| outputSchema | TEXT | NULL | 输出JSON Schema（可选） |
| createdAt/updatedAt/... | - | - | 审计字段 |

3. 索引/约束：
- `UNIQUE(templateId, versionNo)`

### 4. 表：`docgen_job`
1. 业务含义：一次“文档生成任务”的主记录。
2. 字段建议：
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGINT | PK | jobId |
| projectId | BIGINT | NOT NULL | 所属项目 |
| templateVersionId | BIGINT | NOT NULL | 模板版本 |
| businessDescription | TEXT | NOT NULL | 用户输入摘要（可脱敏后） |
| optionsJson | TEXT | NULL | 生成选项（JSON字符串） |
| status | VARCHAR(30) | NOT NULL | PENDING/CLARIFYING/GENERATING/VALIDATING/COMPLETED/FAILED |
| clarifySnapshotJson | TEXT | NULL | 澄清问题与答案快照（可选） |
| finalMarkdownJson | LONGTEXT | NULL | 最终文档集索引/内容摘要（可选） |
| traceId | VARCHAR(64) | NULL | 任务链路traceId |
| errorMessage | TEXT | NULL | 失败原因（可选） |
| createdAt/createdBy/... | - | - | 审计字段 |

3. 索引：
- `idx_docgen_job_projectId_status`(projectId, status)
- `idx_docgen_job_templateVersionId`(templateVersionId)

### 5. 表：`docgen_document`
1. 业务含义：一个任务生成的多份文档（PRD/DESIGN/TECH/FUNC/API/DB/ACCEPT）的落库结果。
2. 字段建议：
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGINT | PK | 文档记录ID |
| jobId | BIGINT | FK | 对应docgen_job.id |
| docType | VARCHAR(20) | NOT NULL | PRD/DESIGN/TECH/FUNC/API/DB/ACCEPT |
| fileName | VARCHAR(200) | NULL | 生成文件名 |
| contentMarkdown | LONGTEXT | NOT NULL | Markdown内容 |
| contentHash | CHAR(64) | NULL | 内容hash（可选） |
| createdAt/createdBy/... | - | - | 审计字段 |

3. 索引/约束：
- `UNIQUE(jobId, docType)`：同一任务同一文档类型只生成一份

### 6. 表：`audit_log`
1. 业务含义：审计日志（安全/追踪）。
2. 字段建议：
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGINT | PK | 日志ID |
| projectId | BIGINT | NULL | 可选关联项目 |
| actorUserId | BIGINT | NULL | 操作用户 |
| action | VARCHAR(100) | NOT NULL | 如 CREATE_JOB/EXPORT/DOCGEN_COMPLETE |
| requestSnapshot | TEXT | NULL | 请求参数摘要 |
| responseSnapshot | TEXT | NULL | 响应摘要（脱敏） |
| traceId | VARCHAR(64) | NULL | 链路traceId |
| createdAt | DATETIME(3) | NOT NULL | 创建时间 |

3. 索引：
- `idx_audit_log_projectId_traceId`(projectId, traceId)

### 7. 选择主键策略说明
- 选择 `BIGINT`（自增或雪花）理由：任务/日志量通常较大，避免整型溢出；联合索引与分库分表更友好。
