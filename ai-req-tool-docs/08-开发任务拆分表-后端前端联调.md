# 开发任务拆分表（后端/前端/联调）

## 1. 排期口径
- 人天定义：1 人全职 1 天。
- 优先级：`P0`（必须先做）、`P1`（核心增强）、`P2`（优化项）。
- 建议节奏：先打通主链路，再补持久化与版本能力。

## 2. 后端任务拆分
| 编号 | 任务 | 优先级 | 预估人天 | 依赖 | 交付物 |
|---|---|---|---:|---|---|
| BE-01 | 任务持久化改造（替换内存 `jobs`） | P0 | 3 | 无 | `job/chat/item/answer` 表落库、查询可用 |
| BE-02 | 聊天接口幂等与重复提交保护 | P0 | 1.5 | BE-01 | 幂等键校验、重复请求防抖 |
| BE-03 | 生成前强校验（`unconfirmedItems` 必空） | P0 | 0.5 | BE-01 | 统一业务错误码与提示 |
| BE-04 | 版本快照接口（保存/列表/详情） | P1 | 2 | BE-01 | 版本 API + 快照入库 |
| BE-05 | 版本差异摘要生成（新增/删除统计） | P1 | 1.5 | BE-04 | diff 摘要字段与返回结构 |
| BE-06 | 审计日志（traceId、action、耗时） | P1 | 1 | BE-01 | 审计表与切面/拦截器 |
| BE-07 | 鉴权接入（基础 JWT） | P2 | 2 | BE-01 | 登录态校验、接口权限基础版 |

后端小计：`12 人天`

前端小计：`10.5 人天`

## 4. 联调与测试任务拆分
| 编号 | 任务 | 优先级 | 预估人天 | 依赖 | 交付物 |
|---|---|---|---:|---|---|
| IT-01 | 创建-澄清-生成-导出全链路联调 | P0 | 1.5 | BE-03, FE-03 | 主流程稳定可演示 |
| IT-02 | 异常链路联调（参数错/状态错/超时） | P0 | 1 | BE-06, FE-06 | 错误行为一致 |
| IT-03 | 版本能力联调（保存/列表/diff） | P1 | 1 | BE-05, FE-05 | 版本链路闭环 |
| IT-04 | 回归测试与验收脚本沉淀 | P1 | 1 | IT-01~03 | 回归清单 + 验收记录 |
| IT-05 | 预发布演练（配置/监控/回滚） | P2 | 0.5 | 全部 | 发布 SOP |

联调小计：`5 人天`

## 5. 总体排期建议
- 总人天：`27.5 人天`
- 建议 2 人并行（1 后端 + 1 前端）：
  - 第 1 周：P0 主链路（BE-01~03 + FE-01~03 + IT-01/02）
  - 第 2 周：P1 版本与审计（BE-04~06 + FE-04~06 + IT-03/04）
  - 第 3 周：P2 鉴权与发布优化（BE-07 + FE-07 + IT-05）

## 6. 里程碑
- 里程碑 M1（第 1 周末）：可稳定演示主链路。
- 里程碑 M2（第 2 周末）：版本管理能力可用。
- 里程碑 M3（第 3 周末）：具备公司内持续优化基础。
