## 2. 设计目标

第一阶段设计目标有 4 个：

1. 支持项目创建前的 AI 对话会话
2. 支持项目资料输入与持久化
3. 支持结构化项目信息与业务知识摘要沉淀
4. 支持从 AI 会话一键创建正式项目

## 3. 总体设计思路

整体采用“项目会话域适配器”的方式实现，而不是直接改造现有需求 AI 工作台。

建议分成 3 层：

1. 项目对话会话层
   - 管理项目创建过程中的对话与资料输入

2. 项目结构化沉淀层
   - 保存项目基础信息、产品信息、业务知识摘要

3. 下游透传层
   - 将项目业务知识摘要透传给需求 AI 工作台

## 4. 数据库设计

### 4.1 扩展 `pm_project`

建议新增字段：

#### `business_knowledge_summary`

- 类型：`TEXT`
- 作用：保存项目级业务知识摘要

#### `creation_mode`

- 类型：`VARCHAR(32)`
- 默认值：`FORM`
- 作用：标识项目来源
- 值建议：
  - `FORM`
  - `AI_CHAT`

建议 SQL：

```sql
ALTER TABLE IF EXISTS pm_project
    ADD COLUMN IF NOT EXISTS business_knowledge_summary TEXT;

ALTER TABLE IF EXISTS pm_project
    ADD COLUMN IF NOT EXISTS creation_mode VARCHAR(32) NOT NULL DEFAULT 'FORM';
```

### 4.2 新增 `pm_project_chat_session`

作用：

- 保存项目创建或项目认知整理过程中的 AI 会话

建议表结构：

```sql
CREATE TABLE IF NOT EXISTS pm_project_chat_session (
    id                         BIGSERIAL PRIMARY KEY,
    project_id                 BIGINT NULL REFERENCES pm_project (id) ON DELETE SET NULL,
    job_id                     VARCHAR(64) NOT NULL,
    status                     VARCHAR(32) NOT NULL,
    assistant_summary          TEXT,
    business_knowledge_summary TEXT,
    structured_info_json       TEXT,
    ready_to_create            BOOLEAN NOT NULL DEFAULT FALSE,
    created_by                 BIGINT NOT NULL REFERENCES sys_user (id),
    created_at                 TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at                 TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE UNIQUE INDEX IF NOT EXISTS uk_pm_project_chat_session_job ON pm_project_chat_session (job_id);
CREATE INDEX IF NOT EXISTS idx_pm_project_chat_session_project ON pm_project_chat_session (project_id);
CREATE INDEX IF NOT EXISTS idx_pm_project_chat_session_status ON pm_project_chat_session (status);
CREATE INDEX IF NOT EXISTS idx_pm_project_chat_session_created_by ON pm_project_chat_session (created_by);
```

字段说明：

- `project_id`
  - 创建前可为空
  - 创建成功后回写正式项目 ID

- `job_id`
  - 对前端透明使用的会话唯一标识

- `status`
  - 建议值：
    - `DRAFT`
    - `ACTIVE`
    - `READY_TO_CREATE`
    - `COMPLETED`
    - `ABANDONED`

- `assistant_summary`
  - AI 当前对项目的阶段性理解

- `business_knowledge_summary`
  - 当前沉淀出的业务知识摘要

- `structured_info_json`
  - 存结构化项目信息快照，便于前端恢复

- `ready_to_create`
  - 是否已达到可以创建正式项目的条件

### 4.3 新增 `pm_project_chat_message`

作用：

- 保存项目 AI 会话中的完整消息历史

建议表结构：

```sql
CREATE TABLE IF NOT EXISTS pm_project_chat_message (
    id            BIGSERIAL PRIMARY KEY,
    session_id    BIGINT NOT NULL REFERENCES pm_project_chat_session (id) ON DELETE CASCADE,
    seq_no        INT NOT NULL,
    role          VARCHAR(16) NOT NULL,
    message_type  VARCHAR(32) NOT NULL,
    content       TEXT NOT NULL,
    created_at    TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE UNIQUE INDEX IF NOT EXISTS uk_pm_project_chat_message_seq ON pm_project_chat_message (session_id, seq_no);
CREATE INDEX IF NOT EXISTS idx_pm_project_chat_message_session ON pm_project_chat_message (session_id);
CREATE INDEX IF NOT EXISTS idx_pm_project_chat_message_role ON pm_project_chat_message (role);
```

字段说明：

- `role`
  - `user`
  - `assistant`
  - `system`

- `message_type`
  - 建议值：
    - `chat`
    - `question`
    - `summary`
    - `material`

### 4.4 新增 `pm_project_source_material`

作用：

- 保存项目资料输入

建议表结构：

```sql
CREATE TABLE IF NOT EXISTS pm_project_source_material (
    id                   BIGSERIAL PRIMARY KEY,
    project_id           BIGINT NULL REFERENCES pm_project (id) ON DELETE SET NULL,
    session_id           BIGINT NULL REFERENCES pm_project_chat_session (id) ON DELETE CASCADE,
    material_type        VARCHAR(16) NOT NULL,
    title                VARCHAR(256),
    source_uri           VARCHAR(1000),
    raw_content          TEXT,
    ai_extracted_summary TEXT,
    created_by           BIGINT NOT NULL REFERENCES sys_user (id),
    created_at           TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at           TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX IF NOT EXISTS idx_pm_project_source_material_project ON pm_project_source_material (project_id);
CREATE INDEX IF NOT EXISTS idx_pm_project_source_material_session ON pm_project_source_material (session_id);
CREATE INDEX IF NOT EXISTS idx_pm_project_source_material_type ON pm_project_source_material (material_type);
```

字段说明：

- `material_type`
  - `URL`
  - `TEXT`

- `source_uri`
  - 存网站地址

- `raw_content`
  - 存用户输入的文本或后续抓取结果

- `ai_extracted_summary`
  - AI 对该资料的摘要提炼

## 6. 接口设计

## 6.1 启动项目 AI 会话

### 接口

`POST /api/projects/ai/conversations`

### 请求体

```json
{
  "projectName": "智能巡检平台",
  "description": "面向园区设备巡检的智能化平台",
  "projectType": "PLATFORM",
  "priority": "P1",
  "materials": [
    {
      "materialType": "URL",
      "title": "官网",
      "sourceUri": "https://example.com"
    },
    {
      "materialType": "TEXT",
      "title": "项目简介",
      "rawContent": "这是一个面向园区设备巡检的智能平台..."
    }
  ]
}
```

### 返回体

```json
{
  "sessionId": 101,
  "jobId": "proj-chat-xxx",
  "assistantMessage": "我已经初步理解该项目...",
  "followUpQuestions": [
    "这个项目的主要客户是谁？",
    "与现有竞品相比，核心差异是什么？"
  ],
  "structuredInfo": {
    "projectBackground": "",
    "similarProducts": "",
    "targetCustomerGroups": "",
    "commercialValue": "",
    "coreProductValue": "",
    "businessKnowledgeSummary": ""
  },
  "readyToCreate": false
}
```

### 说明

- 创建新会话
- 保存初始输入
- 保存资料输入
- 调用 AI 生成首轮回复

## 6.5 从 AI 会话创建正式项目

### 接口

`POST /api/projects/ai/conversations/{sessionId}/create-project`

### 请求体

```json
{
  "projectKey": "AI-INSPECT",
  "projectName": "智能巡检平台",
  "projectType": "PLATFORM",
  "priority": "P1",
  "visibility": "PRIVATE",
  "ownerUserId": 1
}
```

### 返回体

```json
{
  "projectId": 88
}
```

### 说明

- 使用会话中的结构化字段创建 `pm_project`
- 写入 `business_knowledge_summary`
- `creation_mode` 标记为 `AI_CHAT`
- 回写 `pm_project_chat_session.project_id`
- 更新会话状态为 `COMPLETED`

## 7. 状态流转设计

### 7.1 会话状态

建议状态流转：

`DRAFT -> ACTIVE -> READY_TO_CREATE -> COMPLETED`

异常终止时可转：

- `ABANDONED`

### 7.2 `ready_to_create` 判定

第一阶段建议采用简单规则：

满足以下至少 3 项即可为 `true`：

- `projectBackground`
- `targetCustomerGroups`
- `commercialValue`
- `coreProductValue`
- `businessKnowledgeSummary`

也可由 AI 返回 `readyToCreate` 布尔值，再由服务层二次校验。

### 8.2 服务层

### 8.3 AI 层

建议增强：

- `AgentClient`

- `startProjectConversation(...)`

### 8.4 项目域服务

现有：

- `ProjectService`

- 正式项目的创建与维护

也就是说：

- 项目 AI 会话负责“形成项目”
- `ProjectService` 负责“落地项目”

## 11. 结论

从技术实现上，这个方案最关键的是：

1. 复用当前项目主链
2. 参考需求侧会话持久化模式
3. 在项目域新增一套轻量 AI 会话与资料输入体系

第一阶段不需要做复杂 RAG，也不需要做重型知识库。

先把：

- 项目对话
- 资料输入
- 结构化沉淀
- 项目知识摘要透传

这四件事做扎实，就已经能显著提升项目创建和需求澄清质量。
