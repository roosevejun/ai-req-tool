### 2.3 确认当前数据库环境中的向量扩展风险

今天重点验证了 `pgvector` 在当前数据库环境中的可用性。

已确认：

- `pg_available_extensions` 中可以看到 `vector 0.8.0`
- 但 `pg_extension` 中查不到 `vector`
- 一执行 `CREATE EXTENSION IF NOT EXISTS vector;` 就出现数据库节点断连类错误

当前判断是：

- 这不是业务 SQL 语法问题。
- 更像是 Kubernetes 中数据库部署方式、代理层、权限或扩展加载机制存在问题。
- 在问题查清前，不适合把 `vector` 作为当前主链开发的强依赖。
