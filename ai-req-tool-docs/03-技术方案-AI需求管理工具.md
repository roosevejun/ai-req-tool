# 技术方案：AI需求管理工具（与 docgen-webapp 对齐版）

## 1. 现状架构
- 前端：Vue3 + TypeScript + Axios。
- 后端：Spring Boot 3.3 + Java 17。
- LLM 调用：Java `HttpClient` 直连 OpenAI Chat Completions。
- 任务存储：`ConcurrentHashMap<String, JobState>`（内存态）。

## 2. 核心实现映射
- 控制器：`DocGenController`
  - `/jobs` 创建
  - `/jobs/{id}` 查询
  - `/jobs/{id}/chat` 会话
  - `/jobs/{id}/generate` 生成
  - `/jobs/{id}/export` 导出
- 服务层：`DocGenService`
  - `createJob`
  - `chat`
  - `generatePrdFromChat`
  - `loadPrdTemplate`
- AI 客户端：`AgentClient`
  - `startConversation`
  - `continueConversation`
  - `generatePrdFromChat`
  - `callOpenAiForText`

## 3. 配置项（application.yml）
- `agent.template.templateDir`
- `agent.template.templateGlob`
- `agent.llm.openai.baseUrl`
- `agent.llm.openai.model`
- `agent.llm.openai.timeoutMs`

## 4. 输出协议
- 对话输出 JSON：
  - `assistantMessage: string`
  - `confirmedItems: string[]`
  - `unconfirmedItems: string[]`
  - `pendingQuestion: string`
- 文档输出 JSON：
  - `prdMarkdown: string`

## 5. 开发任务拆分建议
1. 后端任务
- 新增持久化层（任务、消息、版本）。
- 把 `JobState` 从内存迁移到 DB。
- 增加生成前校验和幂等保护。
2. 前端任务
- 增加任务列表页与历史检索。
- 增加版本差异详情页。
- 增加草稿恢复提示和冲突提示。
3. 平台任务
- 接入鉴权（JWT/Session）。
- 接入操作审计和调用监控。
