# 验收标准：AI需求管理工具（与现有实现对齐）

## 1. 接口验收
- `POST /api/docgen/jobs` 返回 `jobId` 且 `status` 非空。
- `POST /api/docgen/jobs/{jobId}/chat` 返回完整 `ChatResponse` 字段。
- `POST /api/docgen/jobs/{jobId}/generate` 在 `unconfirmedItems` 非空时应报错。
- `GET /api/docgen/jobs/{jobId}/export` 下载文件名符合 `PRD-{jobId}-revised.md`。

## 2. 状态机验收
- 创建后状态为 `CLARIFYING` 或 `READY`。
- 聊天补充后，`confirmedItems/unconfirmedItems` 会更新。
- `unconfirmedItems` 清空时状态应为 `READY`。
- 生成成功后状态必须为 `COMPLETED`。

## 3. 前端交互验收
- 创建任务后页面显示 `jobId` 与当前 `status`。
- 清单区准确显示“已确认/待确认”。
- `unconfirmedItems.length > 0` 时“生成 PRD”按钮不可用。
- `COMPLETED` 状态显示 `prdMarkdown` 并可导出。

## 4. 错误处理验收
- 业务异常返回 `code=10001`。
- 系统异常返回 `code=20001`。
- 每个错误响应均含 `traceId`。

## 5. 开发任务交付验收（用于拆分）
- 后端任务：
  - 完成任务持久化改造（替换内存 `jobs`）。
  - 完成版本快照接口。
- 前端任务：
  - 完成任务列表页。
  - 完成版本对比页。
- 联调任务：
  - 全链路跑通创建-澄清-生成-导出。
