# 项目管理 + 需求管理设计（对接现有 AI 需求整理能力）

## 1. 目标
- 在现有“AI 对话整理需求 + 生成 PRD”基础上，增加：
- 创建项目
- 项目管理
- 需求管理
- 将现有对话式需求整理能力作为“需求工作台”接入到具体需求项中。

## 2. 页面结构（信息架构）
- `/projects`：项目列表页（项目管理）
- `/projects/create`：创建项目页
- `/projects/:projectId`：项目详情页（成员、统计、需求概览）
- `/projects/:projectId/requirements`：需求列表页（需求管理）
- `/projects/:projectId/requirements/create`：创建需求页
- `/requirements/:requirementId/workbench`：需求工作台（对接现有 DocGen 对话、澄清、PRD 生成）
- `/requirements/:requirementId/versions`：PRD 版本记录页

## 3. 页面设计要点
## 3.1 创建项目页
- 字段：
- 项目名称（必填）
- 项目标识（唯一，如 `AI-REQ`）
- 项目描述（可选）
- 可见范围（私有/组织内）
- 默认负责人（当前登录人）
- 操作：
- 创建并跳转项目详情

## 3.2 项目管理页（项目列表）
- 列：
- 项目名称、项目编码、状态、需求总数、进行中需求数、负责人、更新时间
- 筛选：
- 名称、状态、负责人
- 操作：
- 进入项目
- 编辑项目
- 归档项目

## 3.3 需求管理页（项目下）
- 列：
- 需求标题、优先级、状态、负责人、最近 PRD 版本、更新时间
- 过滤：
- 状态（待分析/澄清中/待评审/已完成）
- 优先级（P0/P1/P2/P3）
- 负责人
- 操作：
- 新建需求
- 进入“需求工作台”
- 查看版本历史

## 3.4 需求工作台（对接现有能力）
- 左侧：
- 需求基础信息卡（标题、状态、优先级、负责人）
- 中间：
- AI 对话区（沿用现有 chat 机制）
- 未确认项卡片（上一题/下一题）
- 右侧：
- PRD 当前版本预览
- 版本时间轴（V1.0/V1.0.1...）
- 操作：
- 发送补充信息
- 生成 PRD
- 发布新版本

## 3.5 版本记录页
- 列：
- 版本号、生成时间、生成人、变更摘要、是否当前版本
- 操作：
- 查看版本内容
- 设为当前版本（可选）
- 导出 markdown

## 4. 与现有 docgen-webapp 的对接方式
- 现有 `jobId` 不再是孤立任务，应绑定到 `requirement_id`。
- 每次生成 PRD 后写入 `requirement_version`。
- 现有“未确认项、待确认问题、chatHistory”落库，支持刷新恢复和协同查看。

## 5. 角色与权限建议
- `ADMIN`：系统配置、用户角色权限
- `PM`：项目/需求全量管理
- `DEV`：查看项目、参与需求澄清
- `REVIEWER`：评审需求并确认版本

权限点示例：
- `PROJECT:CREATE` `PROJECT:EDIT` `PROJECT:ARCHIVE`
- `REQUIREMENT:CREATE` `REQUIREMENT:EDIT` `REQUIREMENT:ASSIGN`
- `REQUIREMENT:AI_CHAT` `REQUIREMENT:GENERATE_PRD` `REQUIREMENT:VERSION_VIEW`

## 6. 状态流建议
- 需求状态：
- `DRAFT`（草稿）
- `CLARIFYING`（澄清中）
- `READY_REVIEW`（待评审）
- `APPROVED`（已确认）
- `DONE`（完成）
- `CANCELLED`（取消）

## 7. 关键数据关系
- 一个项目有多个需求
- 一个需求有多轮 AI 会话
- 一个需求有多个 PRD 版本
- 一个项目有多个成员（成员有项目角色）

## 8. 实施顺序（建议）
1. 先上项目/需求主表 + 列表页（可增删改查）
2. 将现有 DocGen 工作台挂到需求详情
3. 落 PRD 版本表 + 版本页
4. 再补项目成员与权限细化
