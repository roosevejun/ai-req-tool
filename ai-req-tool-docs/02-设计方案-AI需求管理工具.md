# 设计方案：AI需求管理工具（与 docgen-webapp 对齐版）

## 1. 前后端分层
- 前端：单页核心组件 `DocGenPage.vue`。
- 后端：`DocGenController` 暴露 REST API。
- 服务层：`DocGenService` 维护任务状态和流程。
- AI 层：`AgentClient` 负责会话与生成。

## 2. 页面布局（按现有实现）
- 输入区：`businessDescription` + `previousPrdMarkdown`。
- 状态区：`jobId` + `status`。
- 清单区：`confirmedItems` / `unconfirmedItems`。
- 对话区：`chatHistory` + `pendingQuestion`。
- 结果区：`prdMarkdown` + 导出按钮。

## 3. 状态流转
- 创建任务：
  - 输入：`businessDescription`，可选 `previousPrdMarkdown`
  - 输出：`jobId`
  - 状态：`CLARIFYING` 或 `READY`
- 聊天澄清：
  - 输入：`message`
  - 输出：更新 `pendingQuestion` 与确认清单
  - 状态：`CLARIFYING` <-> `READY`
- 文档生成：
  - 条件：`unconfirmedItems` 为空
  - 状态：`READY -> COMPLETED`

## 4. 关键交互规则
- `unconfirmedItems.length > 0` 时禁用“生成 PRD”按钮。
- 每次 `chat` 响应后，前端完整覆盖：
  - `status`
  - `chatHistory`
  - `pendingQuestion`
  - `confirmedItems`
  - `unconfirmedItems`
  - `prdMarkdown`
  - `basePrdMarkdown`
- `COMPLETED` 后仍可查看历史消息，但以导出和复盘为主。

## 5. 异常处理
- 后端统一 `ApiResponse` 包装，异常由 `GlobalExceptionHandler` 捕获。
- 前端优先展示 `HTTP status + body`，便于定位参数和业务错误。
